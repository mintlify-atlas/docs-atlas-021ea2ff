---
title: 'Inventario'
description: 'Manage product inventory across TANCAT locations with role-scoped access.'
---

The inventory API provides endpoints for each physical location. Access is controlled by two permission levels:

- `inventario_sede` — read access, scoped to the employee's assigned location
- `inventario_general` — write access (create, update, delete), available to administrators only

<Note>Requires a valid JWT Bearer token in the Authorization header.</Note>

---

## GET /api/inventario

Returns products for the main location. Results are filtered to the employee's sede unless they have `all: true` (administrator) permission.

```bash
curl --request GET \
  --url https://your-domain.com/api/inventario \
  --header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
```

**Response:**

```json
{
  "success": true,
  "data": [
    {
      "id_producto": 1,
      "nombre": "Pelota de Pádel",
      "stock": 24,
      "id_sede": 1
    }
  ]
}
```

---

## GET /api/inventario/:id

Returns the detail of a single product.

<ParamField path="id" type="number" required>
  Product ID.
</ParamField>

---

## POST /api/inventario

Creates a new product. Requires `inventario_general` permission (administrators only).

---

## PUT /api/inventario/:id

Updates an existing product. Requires `inventario_general` permission.

---

## DELETE /api/inventario/:id

Soft-deletes (deactivates) a product. Requires `inventario_general` permission.

---

## DELETE /api/inventario/:id/purgar

Permanently removes a product. Requires `inventario_general` permission.

<Warning>
  This is a hard delete. The product and all associated data will be permanently removed and cannot be recovered.
</Warning>

---

## GET /api/inventario/categorias/list

Returns product categories. Requires `inventario_sede` permission.

---

## POST /api/inventario/categorias

Creates a new product category. Requires `inventario_general` permission.

---

## GET /api/inventario/sedes/list

Returns the list of sedes. Requires `inventario_sede` permission.

---

## GET /api/inventario/bajo-stock/list

Returns products with stock below their minimum threshold. Requires `inventario_general` permission.

---

## GET /api/inventario/resumen/estadisticas

Returns an inventory summary with totals and category breakdown. Requires `inventario_general` permission.

---

## Error responses

| Status | Condition |
|---|---|
| `401 Unauthorized` | Token is missing or employee is inactive. |
| `403 Forbidden` | Token is expired/invalid, insufficient permissions, or attempting to access a sede you are not assigned to (`SEDE_ACCESS_DENIED`). |
| `500 Internal Server Error` | Database error. |
