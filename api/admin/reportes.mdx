---
title: 'Reportes'
description: 'Retrieve aggregated operational reports for sales, reservations, inventory, occupancy, clients, tournaments, and more.'
---

The reports module exposes multiple dedicated endpoints, each returning a specific category of aggregated data. All endpoints require authentication.

<Note>Requires a valid JWT Bearer token in the Authorization header.</Note>

---

## GET /api/reportes

Returns a list of all available report endpoints.

```bash
curl --request GET \
  --url https://your-domain.com/api/reportes \
  --header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
```

**Response:**

```json
{
  "success": true,
  "data": {
    "disponibles": [
```

## GET /api/reportes/ventas

Sales income report grouped by date or period.

<ParamField query="fecha_desde" type="string">
  Start date in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="fecha_hasta" type="string">
  End date in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="agrupacion" type="string">
  Grouping period for totals. Example: `"dia"`, `"semana"`, `"mes"`.
</ParamField>

```bash
curl --request GET \
  --url 'https://your-domain.com/api/reportes/ventas?fecha_desde=2026-01-01&fecha_hasta=2026-03-31&agrupacion=mes' \
  --header 'Authorization: Bearer ...'
```

## Error responses

| Status | Condition |
|---|---|
| `401 Unauthorized` | Missing, expired, or invalid Bearer token. |
| `403 Forbidden` | Authenticated but insufficient permissions (`PERMISSION_DENIED`). |
| `500 Internal Server Error` | Database query failed. |
