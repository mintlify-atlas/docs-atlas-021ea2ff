---
title: 'Reservas'
description: 'Manage court reservations: list, filter, create, update status, and record payments.'
---

Two route groups manage reservations: `GET/PATCH /api/admin/reservas/*` for admin operations and `POST /api/reservas` for creating new bookings.

---

## GET /api/admin/reservas

Returns a paginated list of reservations with full client, court, and payment details. Pending reservations older than 2 hours are automatically cancelled on each call.

<Note>Requires a valid JWT Bearer token in the Authorization header.</Note>

### Query parameters

<ParamField query="sede" type="string">
  Filter by sede name.
</ParamField>

<ParamField query="deporte" type="string">
  Filter by sport name.
</ParamField>

<ParamField query="estado" type="string">
  Filter by reservation status. Valid values: `pendiente`, `confirmada`, `finalizada`, `cancelada`, `no_presentado`.
</ParamField>

<ParamField query="fecha_desde" type="string">
  Start of date range in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="fecha_hasta" type="string">
  End of date range in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="cliente_busqueda" type="string">
  Search string matched against client name, last name, phone, or DNI.
</ParamField>

<ParamField query="page" type="number">
  Page number for pagination. Defaults to `1`.
</ParamField>

<ParamField query="limit" type="number">
  Results per page. Defaults to `50`.
</ParamField>

```bash

  --url 'https://your-domain.com/api/admin/reservas?estado=pendiente&fecha_desde=2026-03-01&fecha_hasta=2026-03-31' \

```

## PATCH /api/admin/reservas/:id/estado

Updates the status of a reservation. Only valid state transitions are accepted.

<Note>Requires a valid JWT Bearer token in the Authorization header.</Note>

Valid state transitions:

| From | To |
|---|---|
| `pendiente` | `confirmada`, `cancelada` |
| `confirmada` | `finalizada`, `cancelada`, `no_presentado` |
| `finalizada` | — (terminal) |
| `cancelada` | — (terminal) |
| `no_presentado` | — (terminal) |

<ParamField body="estado" type="string" required>
  The new status. Must be a valid transition from the current state.
</ParamField>

```bash
curl --request PATCH \
  --url https://your-domain.com/api/admin/reservas/42/estado \
  --header 'Authorization: Bearer ...' \
  --header 'Content-Type: application/json' \
  --data '{ "estado": "confirmada" }'
```

---

## PATCH /api/admin/reservas/:id/pago-completo

Marks a reservation's payment as fully settled.

<Note>Requires a valid JWT Bearer token in the Authorization header.</Note>

---

## POST /api/admin/reservas/expirar-pendientes

Manually triggers expiry of pending reservations that have passed the time limit. Normally called automatically.

---

## POST /api/admin/reservas/finalizar-vencidas

Automatically finalizes reservations whose time slot has already ended.

---

## POST /api/reservas

Creates a new court reservation. Can be called without authentication for public client bookings, or by an admin on behalf of a client.

<Note>When `creado_por_admin` is `false`, `cliente.dni` is required. Web clients are also limited to 2 reservations per day per DNI or phone number.</Note>

### Request body

<ParamField body="fecha_reserva" type="string" required>
  Date of the reservation in `YYYY-MM-DD` format.
</ParamField>

<ParamField body="id_turno" type="number" required>
  ID of the time slot to book. Must reference an active turno. Returns `409` if already booked for that date.
</ParamField>

<ParamField body="cliente" type="object" required>
  Client details for the reservation.
  <Expandable title="cliente properties" defaultOpen>
    <ParamField body="cliente.nombre" type="string" required>
      Client first name.
    </ParamField>
    <ParamField body="cliente.apellido" type="string" required>
      Client last name.
    </ParamField>
    <ParamField body="cliente.telefono" type="string" required>
      Client phone number.
    </ParamField>
    <ParamField body="cliente.dni" type="string">
      National ID number. Required when `creado_por_admin` is `false`.
    </ParamField>
    <ParamField body="cliente.email" type="string">
      Client email address.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="precio_total" type="number" required>
  Total price of the reservation.
</ParamField>

<ParamField body="sena_pagada" type="number">
  Deposit amount already paid. Defaults to `0`.
</ParamField>

<ParamField body="saldo_pendiente" type="number">
  Remaining balance owed.
</ParamField>

<ParamField body="observaciones" type="string">
  Optional notes or preferences.
</ParamField>

<ParamField body="estado" type="string">
  Initial reservation status. Defaults to `"pendiente"`.
</ParamField>

<ParamField body="creado_por_admin" type="boolean">
  Set to `true` when an admin creates the reservation on behalf of a client. Bypasses the DNI requirement and the 2-reservation-per-day limit. Defaults to `false`.
</ParamField>

### Response

**201 Created**

<ResponseField name="success" type="boolean" required>
  Always `true` on success.
</ResponseField>

<ResponseField name="data" type="object" required>
  <Expandable title="properties" defaultOpen>
    <ResponseField name="id_reserva" type="number" required>
      Unique ID of the newly created reservation.
    </ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Condition |
|---|---|
| `400 Bad Request` | Required fields are missing or `cliente.dni` is absent for a web reservation (`DNI_REQUERIDO`). |
| `404 Not Found` | The specified `id_turno` does not exist or is inactive. |
| `409 Conflict` | The `id_turno` is already booked for `fecha_reserva`. |

### Code example

```bash
curl --request POST \
  --url https://your-domain.com/api/reservas \
  --header 'Content-Type: application/json' \
  --data '{
    "fecha_reserva": "2026-03-20",
    "id_turno": 5,
    "cliente": {
      "nombre": "María",
      "apellido": "López",
      "telefono": "351-123-4567",
      "dni": "38123456",
      "email": "maria@example.com"
    },
    "precio_total": 1500.00,
    "sena_pagada": 500.00,
    "saldo_pendiente": 1000.00,
    "creado_por_admin": false
  }'
```

### Example response

```json
{
  "success": true,
  "data": {
    "id_reserva": 42
  }
}
```
