## GET /api/torneos

Returns all tournaments in the system — both active and inactive — with full details. This is the admin equivalent of the public [GET /api/cliente/torneos](/api/cliente/torneos) endpoint, which returns only active tournaments.

<Note>Requires a valid JWT Bearer token in the Authorization header.</Note>

### Request

No parameters or request body required.

```bash
curl --request GET \
  --url https://your-domain.com/api/torneos \
  --header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
```

### Response

**200 OK**

<ResponseField name="success" type="boolean" required>
  Always `true` on success.
</ResponseField>

<ResponseField name="data" type="array" required>
  Array of all torneo objects, including inactive tournaments.

  <Expandable title="torneo properties">
    <ResponseField name="id_torneo" type="number" required>
      Unique identifier for the tournament.
    </ResponseField>
    <ResponseField name="nombre" type="string" required>
      Display name of the tournament.
    </ResponseField>
    <ResponseField name="fecha_inicio" type="string" required>
      Tournament start date in `YYYY-MM-DD` format.
    </ResponseField>
    <ResponseField name="fecha_fin" type="string" required>
      Tournament end date in `YYYY-MM-DD` format.
    </ResponseField>
    <ResponseField name="deporte" type="string" required>
      Name of the sport for this tournament.
    </ResponseField>
    <ResponseField name="sede" type="string" required>
      Name of the sede hosting this tournament.
    </ResponseField>
    <ResponseField name="activo" type="boolean" required>
      Whether the tournament is currently active.
    </ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Condition |
|---|---|
| `401 Unauthorized` | Missing, expired, or invalid Bearer token. |
