TANCAT's inventory system manages products at each location independently. Stock changes at one sede do not affect another, giving each location manager full control over their own inventory.

## Multi-location inventory

Each location has a dedicated inventory controller and frontend page:

| Location | Frontend page | Controller | API route |
|---|---|---|---|
| Main | `inventario.html` | `inventarioController.js` | `GET /api/inventario` |
| Jacinto RĂ­os | `inventarioJacintoRios.html` | `inventarioJacintoRiosController.js` | `GET /api/inventario/JacintoRios` |
| RincĂłn | `inventarioRincon.html` | `inventarioRinconController.js` | `GET /api/inventario/rincon` |

All three share the same `productos` table in the database, filtered by `id_sede`.

## Productos table structure

Key columns in the `productos` table:

| Column | Type | Description |
|---|---|---|
| `id_producto` | integer | Primary key |
| `nombre` | text | Product name |
| `descripcion` | text | Optional description |
| `precio_unitario` | numeric | Unit selling price |
| `stock` | integer | Current stock quantity |
| `id_sede` | integer | FK to `sedes` â€” determines which location owns this record |
| `id_proveedor` | integer | FK to `proveedores` (optional) |

## Supplier management

Suppliers (`proveedores`) are managed from `frontend/pages/proveedores.html` and backed by `controllers/admin/proveedoresController.js`.

Each supplier record stores contact information and is linked to the products they supply. This relationship enables you to quickly identify the supplier for any product when reordering.

Purchase orders (`compras`) record stock replenishment events:

- Each purchase order references a supplier (`id_proveedor`) and one or more products
- The `comprasController.js` handles creation and listing of purchase orders
- Receiving a purchase order updates the `stock` count for the relevant products at the relevant location

## Access control

Inventory access is scoped by location:

- Staff at Jacinto RĂ­os access `/api/inventario/JacintoRios` only
- Staff at RincĂłn access `/api/inventario/rincon` only
- Superadmin can access all three inventory routes

Attempting to access an inventory route outside your assigned `id_sede` returns a `403 Forbidden` response enforced by `scopeMiddleware.js`.

<Tip>
  To get an aggregate view of stock across all locations, use the Reports section (`/api/reportes`) with the appropriate inventory report type.
</Tip>
