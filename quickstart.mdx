title: "Quickstart"
description: "Get TANCAT System running locally in under 10 minutes."
---

This guide walks you through cloning the repository, connecting a Neon PostgreSQL database, and starting both the backend and frontend services.

## Prerequisites

- Node.js 18 or later
- A [Neon](https://neon.tech) account (free tier is sufficient)
- Git

<Steps>
  <Step title="Clone and install dependencies">
    Clone the repository and run the setup script, which installs dependencies for the root, backend, and frontend packages in one step.

    ```bash
    git clone https://github.com/NicoArmanini/tancat-system.git
    cd tancat-system
    npm run setup
    ```

    The `setup` script runs `node setup-project.js`, which calls `npm run install:all` under the hood — installing packages in the root, `backend/`, and `frontend/` directories.

    <Tip>
      If you ever need to do a clean reinstall, run `npm run clean`. This removes all `node_modules` directories and reinstalls everything from scratch.
    </Tip>
  </Step>

  <Step title="Configure your Neon database">
    Copy the environment variable template and add your Neon connection string.

    ```bash
    cp backend/.env.example backend/.env
    ```

    Open `backend/.env` and fill in the required variables:

    ```env backend/.env
    # Required
    DATABASE_URL=postgresql://user:password@ep-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require
    NODE_ENV=development
    JWT_SECRET=replace-with-a-long-random-string
    JWT_EXPIRES_IN=8h
    CORS_ORIGIN=http://localhost:5173
    ```

    To get your `DATABASE_URL`:

    1. Log in to [Neon console](https://console.neon.tech).
    2. Create a new project (or open an existing one).
    3. Go to **Connection Details** and copy the connection string.
    4. Make sure `sslmode=require` is appended to the URL.

    <Warning>
      Never commit `backend/.env` to version control. The file is already listed in `.gitignore`.
    </Warning>
  </Step>

  <Step title="Initialize the database">
    Run the database setup command to create all tables and seed initial data:

    ```bash
    npm run db:setup
    ```

    This runs migrations followed by seed scripts. You only need to do this once per fresh database.

    <Note>
      To run migrations and seeding independently, use `npm run db:migrate` and `npm run db:seed`.
    </Note>
  </Step>

  <Step title="Start the application">
    Start both the backend and frontend with a single command:

    ```bash
    npm start
    ```

    This runs `node start-tancat.js`, which launches both processes concurrently.

    Once running, you have access to:

    | Service | URL |
    |---|---|
    | Frontend (public portal) | http://localhost:5173 |
    | Backend API | http://localhost:3000 |
    | API documentation | http://localhost:3000/api/docs |
    | Health check endpoint | http://localhost:3000/api/health |

    <Tip>
      To run the backend and frontend in separate terminals during development, use `npm run dev:backend` and `npm run dev:frontend`.
    </Tip>
  </Step>

  <Step title="Verify the health check">
    Confirm the backend is running and connected to the database:

    ```bash
    curl http://localhost:3000/api/health
    ```

    A successful response looks like:

    ```json
    {
      "status": "ok",
      "database": "connected",
      "timestamp": "2026-03-19T00:00:00.000Z"
    }
    ```

    If you see a database connection error, double-check your `DATABASE_URL` in `backend/.env` and ensure `sslmode=require` is present.
  </Step>

  <Step title="Access the admin panel">
    Open the admin login page in your browser:

    ```
    http://localhost:5173/pages/login.html
    ```

    Log in with the credentials created during the seed step. After authentication, you are redirected to the admin dashboard where you can manage reservations, inventory, sales, employees, and tournaments across all three locations.

    <Note>
      The public client portal is available at the root URL: `http://localhost:5173`. Clients can browse locations, check court availability, and view tournaments without logging in.
    </Note>
  </Step>
</Steps>
